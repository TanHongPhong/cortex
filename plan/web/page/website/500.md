# `/500` — Trang lỗi server

**Status:** MVP
**Owner area:** System
**Source of truth:** `plan/web/page_function_matrix.md`, `plan/web/unified_database_schema.md`
**Build decision:** Build

## 1. Mục tiêu

Hiển thị trang thân thiện khi server gặp lỗi, giữ user ở lại thay vì thấy lỗi kỹ thuật.

---

## 2. Trigger

| Trường hợp | Trigger |
|-----------|---------|
| Server error | Lỗi 500 từ server |
| Timeout | Request timeout (30s+) |
| Database error | Không kết nối được DB |
| API error | Internal API failure |
| Build error | Next.js build/runtime error |

---

## 7. Auto-retry Logic

```text
Nếu lỗi là timeout:
→ Hiện countdown "Tự động thử lại sau 10s"
→ Tự động reload sau 10 giây

Nếu lỗi là server error:
→ Không auto-retry
→ Hiện nút "Thử lại" cho user bấm
```

## 9. Error Boundary Integration

```typescript
// Next.js error.tsx
'use client'

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  // Log to Sentry
  useEffect(() => {
    Sentry.captureException(error)
  }, [error])

  return (
    <div className="error-page">
      <h1>500</h1>
      <p>Đã xảy ra lỗi</p>
      <button onClick={reset}>Thử lại</button>
    </div>
  )
}
```

## 11. Different Error Types

| Error Code | Tên | Mô tả | UI |
|-----------|-----|-------|-----|
| 500 | Internal Server Error | Lỗi server chung | "Đã xảy ra lỗi" |
| 502 | Bad Gateway | Gateway error | "Hệ thống đang bảo trì" |
| 503 | Service Unavailable | Server quá tải | "Hệ thống đang bận, thử lại sau" |
| 504 | Gateway Timeout | Timeout | "Hệ thống phản hồi chậm" |
