# `/blog` — Blog / Resources Hub

## 1. Mục tiêu trang

| Mục tiêu         | Mô tả                                                        |
| ---------------- | ------------------------------------------------------------ |
| Lưu trữ tài liệu | Tập trung toàn bộ bài viết, roadmap, template, prompt, guide |
| Kéo traffic      | Dùng SEO để thu hút người học mới                            |
| Tạo niềm tin     | Cho thấy CORTEX có kiến thức và hệ thống học rõ              |
| Thu lead         | Một số tài liệu miễn phí yêu cầu để lại email/Zalo để tải    |
| Hỗ trợ học viên  | Học viên có nơi tra cứu thêm ngoài khóa học                  |

---

# 2. Tên trang nên dùng

Em có thể dùng một trong các tên này:

| Tên             | Nhận xét                      |
| --------------- | ----------------------------- |
| Blog            | Đơn giản, phổ biến            |
| Resources       | Hợp hơn nếu là kho tài liệu   |
| Learning Hub    | Nghe giống trung tâm học tập  |
| AI Resource Hub | Rõ ràng, chuyên nghiệp        |
| CORTEX Library  | Có cảm giác thư viện học liệu |

Anh đề xuất dùng:

```text
/resources hoặc /blog
Tên hiển thị: Resources Hub
```

Nếu vẫn giữ URL `/blog` thì title trên trang nên là **Resources Hub**.

---

# 3. Cấu trúc trang đề xuất

```text
/blog

1. Page Header
2. Featured Resources
3. Resource Categories
4. Resource Library
5. Lead Magnet Section
6. Latest Articles
7. CTA học khóa phù hợp
```

---

# 4. Section 1 — Page Header

**Mục đích:** nói rõ đây là kho tài liệu học AI.

| Thành phần | Yêu cầu                                                            |
| ---------- | ------------------------------------------------------------------ |
| Title      | `Resources Hub` hoặc `Kho tài liệu AI của CORTEX`                  |
| Subtitle   | Giải thích đây là nơi tổng hợp bài viết, roadmap, prompt, template |
| CTA chính  | `Tải roadmap miễn phí`                                             |
| CTA phụ    | `Xem khóa học`                                                     |

Gợi ý nội dung:

```text
Tổng hợp tài liệu học AI, prompt template, hướng dẫn Vibe Coding, AI Agent và automation dành cho người mới bắt đầu.
```

---

# 5. Section 2 — Featured Resources

**Mục đích:** đưa những tài liệu quan trọng lên đầu.

| Loại tài liệu | Ví dụ                                  |
| ------------- | -------------------------------------- |
| Roadmap       | Roadmap học AI Agent từ con số 0       |
| Checklist     | Checklist 30 ngày học Vibe Coding      |
| Prompt Pack   | Bộ prompt học tập/làm việc với AI      |
| Tool Guide    | Hướng dẫn dùng ChatGPT, Claude, Cursor |
| Template      | Template xây AI assistant cá nhân      |

**UI:** 3–4 card lớn, có badge `Featured`, `Free`, `Template`.

---

# 6. Section 3 — Resource Categories

**Mục đích:** phân loại tài liệu để dễ tìm.

| Category           | Nội dung                            |
| ------------------ | ----------------------------------- |
| AI Basics          | Kiến thức nền tảng AI cho người mới |
| Prompting          | Prompt engineering, prompt template |
| Vibe Coding        | Tạo web/app bằng AI coding tools    |
| AI Agent           | Chatbot, assistant, agent workflow  |
| Automation         | Make, n8n, Zapier, workflow tự động |
| Career & Portfolio | Project, CV, portfolio, case study  |
| Tool Guides        | Hướng dẫn dùng các công cụ AI       |

**UI:** category chips hoặc card nhỏ.

---

# 7. Section 4 — Resource Library

**Mục đích:** danh sách toàn bộ tài liệu/bài viết.

Mỗi resource card nên có:

| Thành phần        | Yêu cầu                                            |
| ----------------- | -------------------------------------------------- |
| Title             | Tên tài liệu/bài viết                              |
| Type              | Article / Roadmap / Template / Prompt Pack / Guide |
| Category          | AI Basics, Vibe Coding, Agent...                   |
| Short description | 1–2 dòng                                           |
| Reading time      | Ví dụ: 5 phút đọc                                  |
| Access type       | Free / Email required / Student only               |
| CTA               | `Đọc ngay` / `Tải xuống`                           |

**Có thể có filter nhẹ**, nhưng không cần phức tạp:

```text
All | Articles | Templates | Roadmaps | Prompt Packs | Tool Guides
```

---

# 8. Section 5 — Lead Magnet

**Mục đích:** thu lead từ tài liệu miễn phí.

| Thành phần       | Yêu cầu                                   |
| ---------------- | ----------------------------------------- |
| Tài liệu nổi bật | Ví dụ: “Roadmap học AI Agent từ con số 0” |
| Form ngắn        | Họ tên, email, Zalo/số điện thoại         |
| CTA              | `Nhận tài liệu miễn phí`                  |
| Lưu dữ liệu      | Lưu vào bảng `leads`                      |
| Lead source      | Ghi nguồn là `resource_download`          |

**Không nên bắt form cho tất cả tài liệu.**
Chỉ nên khóa một số tài liệu giá trị cao.

---

# 9. Section 6 — Latest Articles

**Mục đích:** cập nhật nội dung mới nhất.

| Thành phần        | Yêu cầu               |
| ----------------- | --------------------- |
| Danh sách bài mới | 3–6 bài gần nhất      |
| Sort              | Mới nhất lên đầu      |
| CTA               | `Xem tất cả bài viết` |
| SEO               | Mỗi bài có slug riêng |

Ví dụ bài viết ban đầu:

| Bài viết                                        | Mục tiêu                 |
| ----------------------------------------------- | ------------------------ |
| AI Agent là gì?                                 | Giáo dục người mới       |
| Vibe Coding là gì?                              | Giải thích concept chính |
| Học AI nên bắt đầu từ đâu?                      | Dẫn vào Starter/Core     |
| 5 project AI beginner có thể làm                | Dẫn vào Bootcamp         |
| Prompting khác gì với dùng ChatGPT bình thường? | Dẫn vào mini course      |

---

# 10. Section 7 — CTA sang khóa học

**Mục đích:** biến người đọc tài liệu thành người học.

| Thành phần | Yêu cầu                                     |
| ---------- | ------------------------------------------- |
| Headline   | “Muốn học có lộ trình thay vì tự mò?”       |
| Subtext    | CORTEX có workshop, mini course và bootcamp |
| CTA chính  | `Xem khóa học`                              |
| CTA phụ    | `Đăng ký tư vấn`                            |

---

# 11. Trang chi tiết resource `/blog/[slug]`

Sau này mỗi bài/tài liệu nên có trang riêng.

| Thành phần        | Yêu cầu                     |
| ----------------- | --------------------------- |
| Title             | Tiêu đề bài viết            |
| Category/type     | Article, Guide, Template... |
| Author            | CORTEX hoặc tên người viết  |
| Published date    | Ngày đăng                   |
| Reading time      | Thời gian đọc               |
| Content           | Nội dung bài                |
| Download block    | Nếu có file/template        |
| Related resources | Tài liệu liên quan          |
| CTA cuối bài      | Dẫn sang khóa học/contact   |

---

# 12. Yêu cầu chức năng cụ thể

| Nhóm               | Yêu cầu                                                  |
| ------------------ | -------------------------------------------------------- |
| Hiển thị tài liệu  | Lấy từ bảng `resources` hoặc `blog_posts`                |
| Phân loại          | Có category và type                                      |
| Tài liệu tải xuống | Một số tài liệu có file/link tải                         |
| Lead capture       | Tài liệu có `email_required` thì hiện form trước khi tải |
| SEO                | Mỗi bài/tài liệu có title, description, slug, og image   |
| Access control     | Có thể chia Free / Email required / Student only         |
| Search/filter      | Chưa cần mạnh, chỉ cần category tabs là đủ               |
| Admin control      | Admin có thể tạo/sửa/ẩn tài liệu sau này                 |

---

# 13. Data cần có

Nên dùng bảng `resources` thay vì chỉ `blog_posts`, vì em muốn làm kho tài liệu chung.

| Field               | Mục đích                                           |
| ------------------- | -------------------------------------------------- |
| `id`                | ID tài liệu                                        |
| `title`             | Tên bài/tài liệu                                   |
| `slug`              | URL chi tiết                                       |
| `type`              | article / roadmap / template / prompt_pack / guide |
| `category`          | AI Basics / Prompting / Vibe Coding...             |
| `short_description` | Mô tả ngắn                                         |
| `content`           | Nội dung bài                                       |
| `cover_image_url`   | Ảnh cover                                          |
| `download_url`      | Link tải nếu có                                    |
| `access_type`       | free / email_required / student_only               |
| `reading_time`      | Thời gian đọc                                      |
| `status`            | draft / published / hidden                         |
| `created_at`        | Ngày tạo                                           |
| `updated_at`        | Ngày cập nhật                                      |

# 14. UI/UX nên làm

| Phần               | Gợi ý                                                      |
| ------------------ | ---------------------------------------------------------- |
| Tổng thể           | Sạch, giống thư viện tài nguyên                            |
| Card               | Dễ scan, có tag type/category                              |
| Featured resources | Nên nổi bật hơn bài thường                                 |
| Tài liệu tải       | Có icon download/template                                  |
| Mobile             | Card 1 cột, category tabs kéo ngang                        |
| Empty state        | Nếu chưa có tài liệu, hiển thị “Resources are coming soon” |
| Tone               | Hữu ích, thực chiến, không học thuật quá                   |

| Tiêu chí                                        | Đạt / Không |
| ----------------------------------------------- | ----------- |
| Người dùng hiểu đây là kho tài liệu của Academy |             |

| Có category rõ ràng                             |             |
| Có danh sách tài liệu/bài viết                  |             |
| Có CTA tải tài liệu hoặc đăng ký tư vấn         |             |
| Có cơ chế thu lead với tài liệu quan trọng      |             |
| Mỗi resource có trang chi tiết riêng            |             |
| Responsive tốt trên mobile                      |             |
